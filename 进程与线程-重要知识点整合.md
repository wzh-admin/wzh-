# 第一篇章：多线程（从文件15–20梳理）

## 一、创建线程的两种方式

### 方式1：使用 `Thread` 类（函数式）

```python
from threading import Thread

def speak():
    print("说话中...")

t = Thread(target=speak, args=())
t.start()   # 交由操作系统调度
t.join()    # 等待该线程结束
```

**`Thread` 构造参数：**

| 参数 | 说明 |
| --- | --- |
| `group` | 必须为 `None`（保留参数） |
| `target` | 线程要执行的函数 |
| `name` | 线程名称，`None` 则自动分配 |
| `args` | 传给 `target` 的位置参数（元组） |
| `kwargs` | 传给 `target` 的关键字参数（字典） |
| `daemon` | 是否为守护线程（布尔值） |

### 方式2：继承 `Thread` 类（面向对象式）

```python
from threading import Thread

class MyThread(Thread):
    def run(self):          # 重写 run 方法
        print("线程执行中...")

t = MyThread()
t.start()   # start 内部会自动调用 run()
```

**核心要点：**
- 继承 `Thread`，重写 `run()` 方法
- 调用 `start()` 而非直接调用 `run()`，`start()` 会启动新线程并由操作系统调度，`run()` 只在当前线程执行

## 二、线程池 `ThreadPoolExecutor`

### 1. 基本使用

```python
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(3) as executor:    # 最大3个线程
    result = executor.map(work, [1, 2, 3])
    print(list(result))   # 取结果时阻塞
# 离开 with 块自动 shutdown(wait=True)
```

### 2. 获取返回结果的方式

#### a）submit + result

```python
futures = [executor.submit(work, i) for i in range(1, 8)]
executor.shutdown(wait=True)
for f in futures:
    print(f.result())     # 按提交顺序获取结果
```

#### b）as_completed —— 按完成顺序获取

```python
from concurrent.futures import as_completed

futures = [executor.submit(work, i) for i in range(1, 8)]
for f in as_completed(futures):
    print(f.result())     # 谁先完成先处理谁的结果
```

#### c）add_done_callback —— 完成时回调

```python
result_list = []

def done_func(f):
    result_list.append(f.result())

for i in range(1, 8):
    f = executor.submit(work, i)
    f.add_done_callback(done_func)
```

### 3. map 方法批量提交

```python
result = executor.map(work, [1, 2, 3, 4, 5, 6, 7], [lock]*7)
print(list(result))   # 结果顺序与任务顺序一致
```

**`map` 特点：** 不阻塞主线程，立刻返回生成器；真正的阻塞发生在取结果时（如 `list(result)`）；返回结果的顺序与传入的可迭代对象顺序一致。

## 三、GIL（Global Interpreter Lock）全局解释器锁

### GIL 的本质
- GIL 是 CPython 解释器中的一把**互斥锁**
- 作用：无论 CPU 有多少核心，**同一时刻只允许一个线程**执行 Python 字节码

### 重要结论
**CPython 的多线程本质上是并发（快速切换），而不是并行（同时进行）。**

### 为什么要设计 GIL？
为了确保**解释器级别的数据安全**（如引用计数、内存管理）。

### GIL 与业务锁（Lock / RLock）的区别

| 锁类型 | 目标 | 级别 |
| --- | --- | --- |
| GIL | 保护解释器内部数据安全 | 解释器级别 |
| Lock / RLock | 保护业务逻辑不出错 | 应用级别 |

### RLock（可重入锁）示例：卖票系统

```python
from threading import Thread, RLock

current = 1
def sale(lock):
    global current
    while True:
        with lock:
            if current <= 20:
                print(f'出售了第{current}张票！')
                current += 1
            else:
                break
        time.sleep(0.3)

lock = RLock()
t1 = Thread(target=sale, name='窗口1', args=(lock,))
t2 = Thread(target=sale, name='窗口2', args=(lock,))
t1.start()
t2.start()
```

## 四、CPU 密集型 vs IO 密集型的场景选择

### CPU 密集型任务 —— 用多进程

- 特点：大量计算，CPU 一直处于繁忙状态
- GIL 导致多线程无法利用多核优势
- **推荐：多进程（`ProcessPoolExecutor`）**

### IO 密集型任务 —— 用多线程

- 特点：文件读写、网络请求等，CPU 大多时间在等待 IO
- GIL 在 IO 等待时会被释放，线程切换开销小
- **推荐：多线程（`ThreadPoolExecutor`）**

### 选择口诀

> **CPU 密集用多进程，IO 密集用多线程**

# 第二篇章：协程（Coroutine）核心知识点

## 一、协程的概念

### 什么是协程？
协程（Coroutine）是**用户态的轻量级线程**，也叫"微线程"。它不由操作系统调度，而是由**程序自身控制**的——在一个线程内部，通过主动让出和恢复执行来实现"多任务"效果。

### 与线程的对比

| 维度 | 线程 | 协程 |
| --- | --- | --- |
| 调度者 | 操作系统内核 | 程序自身（用户态） |
| 切换开销 | 较大（涉及上下文切换） | 极小（仅改变执行流） |
| 内存占用 | 每个线程有独立栈（数 MB） | 每个协程很小（数 KB） |
| 并发数 | 受系统限制（几千） | 可支持数十万 |
| 是否受 GIL 影响 | 是 | 否（单线程内协作） |

### 协程的本质
协程是一个**可以暂停执行并让出控制权，稍后从暂停处恢复执行的函数**。这一点和生成器（Generator）非常相似——实际上，Python 的协程正是从生成器演变而来。

---

## 二、协程函数与协程对象

### 协程函数（Coroutine Function）

使用 `async def` 定义的函数，称为**协程函数**。

```python
async def hello():
    print("Hello, 协程!")
```

### 协程对象（Coroutine Object）

调用协程函数**不会立即执行**，而是返回一个**协程对象**。

```python
coro = hello()      # 调用函数，返回协程对象，函数体不会执行
print(type(coro))    # <class 'coroutine'>
```

### 执行协程的三种方式

#### 方式1：`await` 直接等待

```python
async def main():
    await hello()    # 等待 hello 执行完毕
```

#### 方式2：`asyncio.run()` 入口

```python
import asyncio
asyncio.run(hello())   # 创建事件循环并运行协程
```

#### 方式3：`asyncio.create_task()` 创建任务

```python
task = asyncio.create_task(hello())
await task
```

## 三、`await` 关键字

### `await` 的作用

`await` 后必须跟一个**可等待对象（Awaitable）**，它做了两件事：
1. **暂停**当前协程的执行
2. **让出控制权**给事件循环，事件循环可以去执行其他协程

```python
import asyncio

async def fetch_data():
    print("开始获取数据...")
    await asyncio.sleep(2)   # 模拟 IO 等待
    print("数据获取完成！")
    return {"data": 123}
```

### 可等待对象（Awaitable）分三种

| 类型 | 说明 |
| --- | --- |
| `coroutine` | 协程对象 |
| `Task` | `asyncio.Task` 对象 |
| `Future` | `asyncio.Future` 对象 |

### `await` 的本质

> `await` 做的事情就是：**我在这里等结果，但不等的时候 CPU 可以去干别的**。

它实现了异步编程的核心——**在等待 IO 时不阻塞线程，让线程去执行其他任务**。

---

## 四、同步执行 vs 异步执行

### 同步执行

多个 `await` 依次执行，前一个完成才执行后一个：

```python
import asyncio

async def task(name, seconds):
    print(f"任务{name}开始...")
    await asyncio.sleep(seconds)
    print(f"任务{name}结束")
    return f"任务{name}的结果"

async def main():
    # 同步执行：总耗时 = 1+2+3 = 6秒
    r1 = await task("A", 1)
    r2 = await task("B", 2)
    r3 = await task("C", 3)
    print(r1, r2, r3)

asyncio.run(main())
```

### 异步执行（并发执行）

使用 `asyncio.create_task()` 将协程包装成任务，让多个任务同时推进：

```python
import asyncio

async def main():
    # 同时创建三个任务
    t1 = asyncio.create_task(task("A", 1))
    t2 = asyncio.create_task(task("B", 2))
    t3 = asyncio.create_task(task("C", 3))

    # 异步执行：总耗时 ≈ 3秒（最长任务）
    r1 = await t1
    r2 = await t2
    r3 = await t3
    print(r1, r2, r3)

asyncio.run(main())
```

## 五、`asyncio.gather` —— 批量异步执行

```python
import asyncio

async def main():
    results = await asyncio.gather(
        task("A", 1),
        task("B", 2),
        task("C", 3),
    )
    print(results)   # 结果按传入顺序返回

asyncio.run(main())
```

### `gather` 特点
- 同时启动所有协程
- 返回结果的顺序与传入顺序一致（不是完成顺序）
- 总耗时 ≈ 最慢的那个任务
- 如果某个协程抛出异常，`gather` 默认会向上传播

---

## 六、`asyncio.wait` —— 更灵活的任务等待

```python
import asyncio

async def main():
    tasks = [
        asyncio.create_task(task("A", 1)),
        asyncio.create_task(task("B", 2)),
        asyncio.create_task(task("C", 3)),
    ]

    # 等待所有任务完成
    done, pending = await asyncio.wait(tasks, return_when=asyncio.ALL_COMPLETED)
    for t in done:
        print(t.result())

asyncio.run(main())
```

### `return_when` 参数

| 取值 | 含义 |
| --- | --- |
| `ALL_COMPLETED` | 等待所有任务完成（默认） |
| `FIRST_COMPLETED` | 只要有第一个完成就返回 |
| `FIRST_EXCEPTION` | 第一个异常产生时返回 |

## 七、综合对照示例

### 多线程版（IO密集型任务）

```python
from concurrent.futures import ThreadPoolExecutor
import time

def io_task(name):
    print(f"任务{name}开始...")
    time.sleep(1)   # 模拟 IO 操作
    print(f"任务{name}结束")
    return f"结果{name}"

with ThreadPoolExecutor(3) as ex:
    futures = [ex.submit(io_task, i) for i in range(1, 4)]
    for f in futures:
        print(f.result())
```

### 协程版（异步执行）

```python
import asyncio

async def io_task(name):
    print(f"任务{name}开始...")
    await asyncio.sleep(1)   # 模拟 IO 操作，不阻塞线程
    print(f"任务{name}结束")
    return f"结果{name}"

async def main():
    tasks = [asyncio.create_task(io_task(i)) for i in range(1, 4)]
    results = await asyncio.gather(*tasks)
    print(results)

asyncio.run(main())
```

**关键区别：** 多线程版背后有多个线程（受操作系统调度），而协程版只有一个线程（由事件循环调度，切换完全在用户态完成）。

## 八、总结对比：进程、线程、协程

| 特性 | 进程 | 线程 | 协程 |
| --- | --- | --- | --- |
| 调度者 | 操作系统 | 操作系统 | 程序自身（事件循环） |
| 切换开销 | 最大 | 中等 | 最小 |
| 内存隔离 | 独立 | 共享 | 共享 |
| 可并行（多核） | 是 | 受 GIL 限制 | 否（单线程内） |
| 适合场景 | CPU 密集型 | IO 密集型 | IO 密集型（超高并发） |
| 并发数量级 | 几十 ~ 几百 | 几百 ~ 几千 | 几万 ~ 几十万 |

### 选择指南

1. **CPU 密集型任务**（大量计算、数值运算、视频编码等）→ **多进程**
2. **IO 密集型任务**（网络请求、文件读写、数据库操作等）
   - 并发量不高 → **多线程**
   - 并发量极高（如 Web 服务器处理上万个连接）→ **协程**
3. **需要强数据隔离、稳定性要求高** → **多进程**
