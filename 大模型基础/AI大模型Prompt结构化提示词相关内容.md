## 1. Prompt（提示词）是什么？

### 🤔 是什么？

**Prompt（提示词）** 就是你向AI提出的问题或指令。就像你给一个新员工交代任务——"帮我把这份资料翻译成英文"、"写一篇关于机器学习的500字介绍"——这些都属于Prompt。

### 🎯 为什么需要学习写Prompt？

- AI不能读心，它的输出质量**完全取决于你的输入**
- 好的Prompt = 一次说清楚，结果一步到位
- 差的Prompt = 来回扯皮，浪费时间

### 🏋️ 一个比喻

`
把AI想象成一位能力超强但需要明确指令的实习生：
❌ "帮我把这个弄一下。" → AI一脸懵
✅ "请把这篇中文文章翻译成英文，保持专业语气，并标注出专业术语。" → 完美执行
`

## 2. 提示词基本技巧

### 2.1 清晰具体（Be Specific）

| ❌ 模糊 | ✅ 清晰 |
|:---|:---|
| "写一篇关于猫的文章" | "写一篇500字的科普短文，介绍中国狸花猫的外形特征和性格特点，面向宠物爱好者" |
| "帮我改一下这段文字" | "请把这段文字改得更正式，适合用在商务邮件中，保留核心信息" |
| "画一个图表" | "用Python的matplotlib画一个柱状图，展示A组和B组在三个季度的销售对比数据" |

### 2.2 设定角色（Role Prompting）

给AI一个**身份设定**，让它在特定专业领域内回答：

`
✅ "你是一位资深Python工程师，请解释Python装饰器的原理和使用场景。"
✅ "你是一位小学语文老师，请用简单的语言解释什么是'比喻句'。"
✅ "你是一位营养师，请根据以下食材搭配一份适合减肥人群的一日三餐。"
`

**为什么有效？** 角色设定激活了AI在该领域的知识分布，回答会更专业、更聚焦。

### 2.3 提供示例（Few-Shot Prompting）

给AI一个或多个**范例**，让它照着样式输出：

`
请按照以下格式回答。示例：
问：什么是资本？
答：资本是用来生产其他商品的生产要素，包括资金、设备和厂房。

问：什么是边际效用？
答：
`

### 2.4 分步骤引导（Chain of Thought）

让AI**一步步推理**，而不是直接跳结论：

`
❌ "从北京到上海坐高铁要多久？" → 可能得到不准确答案
✅ "请问从北京到上海的高铁，总里程约1300公里，高铁时速约300km/h，
请分步骤计算需要的时间，包括中途停站的20分钟。" → 推理过程更可靠
`

### 2.5 设定输出格式（Format Control）

明确指定输出的结构：

`
请列出5种常见的机器学习算法，按以下Markdown表格格式输出：
| 算法名称 | 类别 | 一句话描述 |
`

### 📊 技巧对比表

| 技巧 | 一句话说明 | 适用场景 |
|:---|:---|:---|
| **清晰具体** | 把你想说的细节都写上 | 所有场景，最基础也最重要 |
| **角色设定** | 给AI一个专业身份 | 需要专业知识或特定语气时 |
| **提供示例** | 给1-3个范例照着做 | 复杂格式、创意写作 |
| **分步引导** | 让AI一步步推理 | 数学题、逻辑推理、分析任务 |
| **格式控制** | 指定Markdown/列表/表格等 | 需要结构化输出时 |

## 3. Top P 参数（核采样）

### 🤔 是什么？

**Top P**（也叫核采样，nucleus sampling）是控制AI生成文本**随机性**的一个参数。

- 取值范围：0 到 1
- 默认值通常是 0.9 或 1.0
- 值越大 → 输出越**多样、有创意**
- 值越小 → 输出越**确定、保守**

### 🎯 它怎么工作？

`
想象AI脑子里有一个"候选词"的排行榜，按可能性从高到低排列：

第1名: "猫"     概率 25%
第2名: "狗"     概率 20%
第3名: "兔子"   概率 15%
第4名: "仓鼠"   概率 10%
第5名: "金鱼"   概率 8%
第6名: "乌龟"   概率 6%
第7名: "鹦鹉"   概率 5%
第8名: "蜥蜴"   概率 4%
...

- Top P = 0.9 → AI从累计概率达到90%的"候选池"里选
                （猫+狗+兔子+仓鼠+金鱼+乌龟+鹦鹉 = 89% ≈ 90%，蜥蜴太冷门了不考虑）
- Top P = 0.3 → AI只从累计概率30%的极小池里选（几乎总是选"猫"）
- Top P = 1.0 → AI把All候选都纳入考虑（但不一定选到冷门的，因为概率还是低）
`

### 🏋️ 一个比喻

`
Top P 就像你在餐厅点菜：
- Top P = 0.1：只看菜单上最流行的前3道菜（稳定不出错）
- Top P = 0.5：看前10道热门菜（有选择但不会太离谱）
- Top P = 0.9：看菜单上几乎所有菜（偶尔会尝试奇怪的搭配）
- Top P = 1.0：把菜单翻个底朝天（可能选到极其冷门的菜）
`

### 📊 Temperature vs Top P

| 参数 | 作用 | 值小 | 值大 |
|:---|:---|:---|:---|
| **Temperature（温度）** | 压扁或拉高概率分布 | 更确定、重复性高 | 更随机、创意多 |
| **Top P（核采样）** | 限制候选词范围 | 只选最高概率的少数词 | 考虑更多候选词 |

**实际使用建议：** 一般调整其中一个即可。推荐优先调 **Temperature**，Top P保持0.9不变。

## 4. 结构化提示词 ⭐ 重点

### 🤔 是什么？

**结构化提示词**就是把Prompt按照一套**固定的模板格式**来组织，让AI能清晰理解你的需求。就像填表格一样——每个字段对应一个需求维度。

### 🎯 为什么它是重点？

- 让复杂需求**一次说清**，减少来回修改
- 输出可预测、可重复（尤其是给AI工具写Prompt时）
- 团队协作时可以**复用模板**
- 是**高级AI使用者**的标志性技能

### 4.1 经典结构化模板

`
# 角色
你是一位[身份描述]

# 任务
你需要完成：[具体任务描述]

# 背景
[提供必要的上下文信息]

# 要求
1. [格式要求]
2. [语气要求]
3. [长度限制]
4. [其他约束]

# 输出格式
[指定输出的结构，如：Markdown表格、JSON、分点列表等]
`

### 4.2 完整示例

`
# 角色
你是一位资深Python工程师和数据分析师

# 任务
帮我写一个Python爬虫脚本，从某个静态网页抓取新闻标题和发布时间

# 背景
目标网页是典型的CMS新闻列表页，内容直接包含在HTML中，无需JavaScript渲染

# 要求
1. 使用 requests + BeautifulSoup 实现
2. 添加基本的错误处理和超时设置
3. 输出为CSV文件
4. 添加必要的代码注释

# 输出格式
直接输出完整的Python代码，不需要额外解释
`

### 4.3 更精细的结构（COAST框架）

`
C - Context（上下文）：描述背景情况和约束
O - Objective（目标）：明确要达到什么目的
A - Action（行动）：具体要做什么
S - Scenario（场景）：在什么场景下使用
T - Tone / Task（语气/任务）：期望的风格和语气
`

### 4.4 另一种实用框架（CRISPE）

`
C - Capacity（能力）：给AI设定角色
R - Insight（洞察）：提供背景和目标
S - Statement（陈述）：明确指令
P - Personality（个性）：指定风格
E - Experiment（实验）：要求提供多个方案
`

### 📊 结构化框架对比

| 框架 | 核心字段 | 适合场景 |
|:---|:---|:---|
| **基础结构化** | 角色+任务+背景+要求+输出格式 | 日常使用，通用性强 |
| **COAST** | 上下文+目标+行动+场景+语气 | 商务写作、项目沟通 |
| **CRISPE** | 能力+洞察+陈述+个性+实验 | 创意写作、多方案生成 |

### 💡 结构化提示词的最佳实践

1. **先写角色**：明确告诉AI"你是谁"，让回答更专业
2. **再写任务**：一句话说清楚要做什么
3. **补充背景**：让AI理解"为什么"要做这件事
4. **约束条件放最后**：格式、长度、风格等限制
5. **给例子最好**：1-2个示例胜过千言万语
6. **每次只用一种框架**：别混用，保持结构清晰

## 5. 用学习法写提示词（费曼、番茄、SQ3R）

把这些经典学习方法反过来**用在提示词上**，让AI帮你更高效地学习。

### 5.1 费曼学习法 + Prompt

#### 🤔 什么是费曼学习法？

核心思想：**用最简单的语言解释复杂概念**，如果能讲给小学生听，说明你真懂了。

#### 🎯 如何用Prompt实现？

`
# 角色
你是一位能用最简单语言解释复杂概念的老师

# 任务
请用费曼学习法帮我理解"Transformer"这个AI概念

# 要求
1. 先用一句话说明核心思想
2. 用日常生活中的比喻来解释
3. 不要使用专业术语，如果必须用请先解释
4. 最后用一段话总结，让10岁小孩也能听懂
`

**应用场景**：学习任何新概念时，让AI用费曼法给你讲解。

### 5.2 番茄工作法 + Prompt

#### 🤔 什么是番茄工作法？

- 工作25分钟 → 休息5分钟 → 循环
- 每4个番茄钟后休息15-30分钟
- 专注一件任务，不打断

#### 🎯 如何用Prompt实现？

`
# 角色
你是一位学习规划师

# 任务
帮我制定一个基于番茄工作法的学习计划，主题是"Python爬虫入门"

# 内容
将学习内容拆分为6个番茄钟（每个25分钟），每个番茄钟聚焦一个子课题

# 要求
1. 每个番茄钟的具体学习内容
2. 5分钟休息时间做什么
3. 每2个番茄钟后的学习效果自测题
`

**应用场景**：制定学习计划、分解复杂任务。

### 5.3 SQ3R 学习法 + Prompt

#### 🤔 什么是SQ3R？

| 步骤 | 英文 | 含义 |
|:---|:---|:---|
| **S** | Survey | 浏览 → 先看标题、目录、图表，了解整体结构 |
| **Q** | Question | 提问 → 把标题转成问题，带着问题读 |
| **R1** | Read | 阅读 → 带着问题仔细读内容 |
| **R2** | Recite | 复述 → 合上书，用自己的话回答 |
| **R3** | Review | 复习 → 定期回顾，巩固记忆 |

#### 🎯 如何用Prompt实现？

`
# 角色
你是SQ3R学习法的引导老师

# 任务
请用SQ3R方法帮我来学习[主题：机器学习中的过拟合问题]

# 流程
步骤1 - Survey（浏览）：给我列出这个主题有哪几个关键子话题
步骤2 - Question（提问）：针对每个子话题给我3个引导性问题
步骤3 - Read（阅读）：依次解答这些问题，每个答案不超过100字
步骤4 - Recite（复述）：请检查我对核心概念的理解，出一段填空或判断对错题
步骤5 - Review（复习）：最后给我一张核心概念的快速回顾卡片（key cards）
`

**应用场景**：系统性地学习一个新领域的理论知识。

### 📊 三种学习法对比

| 方法 | 核心思想 | Prompt最佳用途 |
|:---|:---|:---|
| **费曼学习法** | 用最简单的语言解释 | 理解新概念、检查自己的理解 |
| **番茄工作法** | 短时间高度专注+休息 | 制定学习/工作计划 |
| **SQ3R** | 先浏览再提问再阅读再复述再复习 | 系统学习一个领域的理论知识 |

## 6. Anaconda / Jupyter 入门

### 6.1 Anaconda 是什么？

**Anaconda** 是一个Python的**发行版**，预装了数据科学常用的几百个库。不用自己一个个去装包。

#### 🎯 为什么要用Anaconda？

| 问题 | 解决方案 |
|:---|:---|
| 装Python后还得自己装numpy/pandas/matplotlib... | Anaconda自带这些 |
| 不同项目需要的库版本不同 | Anaconda支持创建**虚拟环境**，项目之间隔离 |
| 包依赖冲突 | conda包管理器自动解决依赖 |

### 6.2 核心命令速查

`
# 创建一个新环境
conda create -n myenv python=3.10

# 激活环境
conda activate myenv

# 安装包
conda install pandas numpy matplotlib jupyter

# 查看已有环境
conda env list

# 退出环境
conda deactivate

# 删除环境
conda remove -n myenv --all
`

### 6.3 Jupyter Notebook 是什么？

Jupyter是一个**交互式编程环境**，可以在浏览器里写代码、看结果、写笔记，全部在一个文件里完成。

#### 🏋️ 一个比喻

`
普通Python文件：就像做菜时写一张完整的菜谱（写好全部再执行）
Jupyter Notebook：像边做菜边试味道（写一行→看结果→写下一行→再看结果）

对于学习Python的人来说，Jupyter简直是神器！
`

### 6.4 Jupyter 基本操作

`
# 安装（已装Anaconda则自带）
conda install jupyter

# 启动
jupyter notebook
# 或新版
jupyter lab

# 常用快捷键
- Shift + Enter：运行当前单元格
- A：在当前单元格上方插入新单元格
- B：在当前单元格下方插入新单元格
- DD：删除当前单元格
- M：将单元格切换为Markdown（写注释/笔记）
- Y：将单元格切换为代码模式
`

### 6.5 学习Python的好路线

`
1. 装Anaconda（一步到位）
2. 启动Jupyter Notebook
3. 创建一个新的.ipynb文件
4. 在代码格子里写 Python 代码 → Shift+Enter运行
5. 在Markdown格子里写笔记
6. 一个Notebook = 代码 + 结果 + 笔记
`

## 7. Python 爬虫基础

### 7.1 什么是爬虫？

**爬虫（Web Scraping）** 就是从网页上批量提取数据。比如：收集某电商网站的商品价格、抓取新闻标题、下载图片等。

### 7.2 爬虫的合法性和道德

> ⚠️ 重要提醒：爬虫一定要遵守robots.txt协议、不要高频率请求（会被封IP）、不要爬取有版权的数据、不要爬取需要登录才能访问的私人内容。

### 7.3 核心工具

| 库 | 用途 | 类比 |
|:---|:---|:---|
| **requests** | 发送HTTP请求，获取网页HTML | 像浏览器发送一个"请求" |
| **BeautifulSoup** | 解析HTML，提取想要的数据 | 像在网页上"选中"元素 |
| **Scrapy** | 完整的爬虫框架（适合大规模） | 像一个全自动爬虫工厂 |
| **Selenium** | 模拟真实浏览器（处理JS渲染的页面） | 像遥控器控制浏览器 |

### 7.4 最简单的爬虫示例

`python
# 安装：pip install requests beautifulsoup4

import requests
from bs4 import BeautifulSoup

# 1. 请求网页
url = "https://example.com/news"
response = requests.get(url, timeout=10)
response.encoding = "utf-8"

# 2. 解析HTML
soup = BeautifulSoup(response.text, "html.parser")

# 3. 提取数据（以提取所有新闻标题为例）
titles = soup.find_all("h2", class_="news-title")
for i, title in enumerate(titles, 1):
    print(f"{i}. {title.text.strip()}")
`

### 7.5 爬虫常见坑和解决

| 问题 | 原因 | 解决方法 |
|:---|:---|:---|
| 返回403错误 | 网站禁止爬虫 | 添加请求头：headers = {"User-Agent": "..."} |
| 获取不到数据 | 页面是JS动态渲染 | 改用 Selenium 或分析API接口 |
| 请求太频繁被封 | 短时间内请求过多 | 加 	ime.sleep(1) 降低频率 |
| 中文乱码 | 编码不一致 | 设置

在Jupyter中写爬虫非常适合——可以一步步调试：

`
单元格1：写requests请求，看返回的状态码
单元格2：打印HTML的前500个字符，确认内容正确
